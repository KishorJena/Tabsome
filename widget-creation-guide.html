‚ú® Best Practices

  • Keep it simple: Focus on one main piece of information or functionality
  • Use relative units: Use %, em, or rem instead of fixed pixels for better responsiveness
  • Test thoroughly: Preview your widget before publishing to ensure it works perfectly
  • Optimize performance: Avoid heavy animations or excessive API calls that might slow down the new tab
  • Consider readability: Use sufficient contrast and legible fonts (minimum 14px for body text)
  • Handle errors gracefully: If fetching external data, show a friendly message if it fails
  • Respect privacy: Don't collect or transmit user data without explicit permission
  • Make it accessible: Use semantic HTML and provide alt text for images
Important: Widgets run in a sandboxed iframe for security. They cannot access the parent page, local storage is isolated, and cross-origin requests need proper CORS headers.

Every Tabsome widget consists of three components:

Widget JSON Format 
{
  "id": "unique-widget-id",
  "title": "My Awesome Widget",
  "description": "A cool widget that does amazing things",
  "labels": ["productivity", "quotes", "custom"],
  "widget": {
    "html": "<div class=\"widget\">...</div>",
    "css": ".widget { ... }",
    "js": "// Your JavaScript code here"
  }
}
Note: The html, css, and js fields should contain your widget's code as strings. Special characters need to be properly escaped in JSON.

üöÄ Advanced Tips

Using External APIs

You can fetch data from external APIs, but remember:

  • The API must support CORS (Cross-Origin Resource Sharing)
  • Use HTTPS endpoints only
  • Handle loading states and errors gracefully
  • Consider caching to reduce API calls

Performance Optimization

  • Minimize DOM manipulation
  • Use CSS animations instead of JavaScript when possible
  • Debounce frequent updates (like countdown timers)
  • Lazy load heavy resources

Styling Tips

  • Use CSS variables for easy theme customization
  • Implement smooth transitions for a polished feel
  • Consider using flexbox or grid for layout
  • Test with different background images (the new tab has custom backgrounds)
